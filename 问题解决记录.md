### 问题 1：AI服务恢复后仍返回错误信息

#### 📋 问题描述

**现象**：Python服务因网络问题暂时不可用，返回错误信息"AI服务暂时不可用"。服务恢复后，用户继续提问时：
1. 回答仍然显示"AI服务暂时不可用"
2. 返回速度很快（毫秒级）
3. 即使切换新会话也可能出现相同问题

**影响范围**：所有用户对话功能

**严重程度**：高

---

#### 🔍 原因分析

经排查，发现**两个叠加的Bug**：

##### Bug 1：对话历史污染
- 当服务不可用时，错误信息"AI服务暂时不可用"被保存到对话历史中
- 后续问题携带这个历史上下文发送给LLM
- LLM看到对话历史中有"服务不可用"的记录，就顺着这个思路回答

##### Bug 2：Redis缓存污染
- 错误响应被缓存到Redis中（缓存键格式：`ai:answer:{userId}:{question}`）
- 服务恢复后，相同问题直接读取缓存的错误回答
- 导致即使服务恢复，用户仍然看到错误信息

**问题流程图**：
```
用户提问 → Redis缓存命中 → 返回缓存的错误回答
       ↓
（服务已恢复，但缓存未更新）
```

---

#### 🛠️ 解决方案

##### 方案概述
采用"双重防护"策略：
1. **Python端**：过滤对话历史中的错误信息，防止LLM被误导
2. **Java端**：改进缓存策略，不缓存/不使用错误响应

##### 具体修复

**1. Python端修复（`python-service/core/llm.py`）**

新增对话历史清理函数：
```python
def clean_conversation_context(self, context: str) -> str:
    """清理对话上下文，移除错误信息"""
    error_keywords = [
        "AI服务暂时不可用",
        "服务不可用", 
        "系统错误",
        "无法连接",
        "网络错误",
        "超时",
        "API密钥",
        "配置错误"
    ]
    # 过滤包含错误关键词的行
    lines = context.split("\n")
    cleaned_lines = [line for line in lines 
                    if not any(keyword in line for keyword in error_keywords)]
    return "\n".join(cleaned_lines)
```

更新Prompt模板，添加规则：
```python
self.prompt = PromptTemplate.from_template("""
重要规则：
4. 不要提及"AI服务不可用"、"系统错误"等技术问题，你始终处于正常工作状态
...
""")
```

**2. Java端修复（`AiServiceImpl.java`）**

新增错误响应检测：
```java
private boolean isErrorResponse(String answer) {
    String[] errorKeywords = {
        "AI服务暂时不可用",
        "服务不可用",
        "系统错误",
        "无法连接",
        "网络错误",
        "暂时无法回答",
        "稍后再试"
    };
    for (String keyword : errorKeywords) {
        if (answer.contains(keyword)) {
            return true;
        }
    }
    return false;
}
```

新增友好错误提示语：
```java
private String getFriendlyErrorMessage(String errorType) {
    String[] friendlyMessages = {
        "正在努力处理您的问题，请稍等片刻...",
        "系统正在维护中，很快就会恢复，请稍后再试",
        "服务暂时繁忙，请稍后重试",
        "我正在连接知识库，请耐心等待",
        "让我想想，马上为您解答"
    };
    int index = Math.abs(errorType.hashCode()) % friendlyMessages.length;
    return friendlyMessages[index];
}
```

修改缓存逻辑：
```java
// 缓存命中时：如果是错误响应，不使用缓存
if (cachedResponse != null) {
    if (isErrorResponse(cachedResponse.getAnswer())) {
        log.info("Cache hit but contains error response, refreshing...");
        // 不使用缓存，继续执行请求
    } else {
        return cachedResponse;
    }
}

// 缓存写入时：错误响应不缓存
if (!isErrorResponse(answer)) {
    cacheService.set(CacheConfig.CACHE_AI_ANSWER, cacheKey, aiResponse);
}
```

### 问题 2：AI回答引用文档不一致问题

#### 📋 问题描述

**现象**：AI回答有时会引用不相关的文档作为参考来源，有时又不返回任何参考来源，存在以下表现：
1. 回答"一级封锁协议"时正确返回了相关文档《数据库系统概论速成》
2. 回答"三级封锁协议"时没有返回任何参考来源
3. 有时会返回与问题无关的文档

**影响范围**：知识问答功能的可信度和准确性

**严重程度**：🟡 中

---

#### 🔍 原因分析（待确认）

可能的原因包括：

1. **向量检索相关性问题**
   - 向量相似度计算可能返回不相关的文档
   - 文档切分质量影响检索结果
   - 词嵌入模型对某些术语的理解不准确

2. **检索阈值设置**
   - 相似度阈值过高导致漏检
   - 阈值过低导致误检

3. **LLM 引用逻辑**
   - LLM 可能在没有参考文档的情况下生成回答
   - 参考来源与回答内容的映射逻辑可能存在问题

4. **文档内容覆盖**
   - 知识库中文档内容可能不完整或存在冲突
   - 相同主题的文档可能分布在多个文件中
